# 线下送样并行支线触发与验收模板

> 适用范围：`紫铜手冲/咖啡壶同品`
>
> 固定前提：
> - 它不是银内壁款
> - 当前阶段不下单、不支付样品费
> - 不新增询盘、不新增供应商搜索
> - 该支线只作为线下并行流程，不阻塞主线

## 1. 支线触发表

| 触发场景 | 触发前必须已确认的前提 | 触发人 | 支线动作 | 主线是否等待 |
| --- | --- | --- | --- | --- |
| CEO 明确拍板“允许启动线下送样支线” | 已确认唯一保留方向仍为 `紫铜手冲/咖啡壶同品`；已确认“非银内壁款”表述不变；已确认当前主线继续按零送样推进 | CEO | 允许进入线下协调准备，不代表立刻下样 | 否 |
| 用户或 CEO 明确要求“需要线下实物判断控流/手感/做工” | 已完成现有图文资料、规格、包装、风险判断；主线资料整理已可继续推进 | CEO 或运营总监 | 记录“为什么需要样品”与预期验证点，建立送样任务卡 | 否 |
| 已有内部线下资源可零额外成本拿到观察样 | 不新增供应商池；不重开询盘；样品来源仍限于已锁定方向的既有渠道 | 供应链协调 | 建立样前证据包与收样责任人，准备到样验收 | 否 |
| 主线进入需要实物解锁的节点 | 仅限需要实物的动作仍处于锁定状态，例如实拍图、重量复核、最终上架解锁 | 运营总监 | 将技术/运营待解锁项挂到样品回传清单 | 否 |

### 触发前不满足则不得进入支线

- CEO 未明确拍板，不进入。
- 方向不再是 `紫铜手冲/咖啡壶同品`，不进入。
- 需要通过新增询盘、扩供应商池、追加研究才能推进，不进入。
- 需要先支付样品费、运费或下单才能推进，不进入。

## 2. 触发前不做什么

在 CEO 未拍板前，以下动作全部禁止：

- 不下单。
- 不支付样品费。
- 不支付国内运费、国际运费或任何名义的送样成本。
- 不扩供应商池。
- 不重开询盘。
- 不新增 1688 搜索、以图搜货或额外比价。
- 不把该产品改写成“银内壁款”。
- 不把线下送样结论当作主线前置门槛。

## 3. 线下送样执行包

### 3.1 到样前证据清单

在样品真正到手前，必须先留存以下证据，避免后续无法追责或无法复盘：

| 证据项 | 必留内容 | 责任人 |
| --- | --- | --- |
| 样品身份确认 | 样品对应 SKU/产品名、容量、颜色/表面处理、是否仍为 `紫铜手冲/咖啡壶同品` | 供应链协调 |
| 已锁定前提截图 | “非银内壁款”确认记录；当前阶段不付费、不下单的决策记录 | 供应链协调 |
| 来源证明 | 现有资料页、历史沟通记录、既有商品链接或图册页，不新增外部搜索 | 供应链协调 |
| 规格基线 | 尺寸、容量、重量、材质、包装方式的现有记录 | 供应链协调 |
| 包装基线 | 单品包装方式、缓冲材料、外箱信息或既有包装照片/说明 | 供应链协调 |
| 样前视频/图片 | 壶嘴、壶口、焊点、内壁、底部、把手、包装的实拍 | 供应链协调 |
| 尺量/称重记录 | 尺寸与重量视频或照片，作为到样后复核基线 | 供应链协调 |
| 验收负责人 | 收样人、验收人、回传人姓名/角色与预计时间 | 运营总监 |

### 3.2 到样后验收记录模板

以下模板继承 `HTH-216` 已固定的观察样验收口径，只用于这把紫铜手冲壶，不扩展到其他方向。

```md
# 观察样验收记录

- 样品名称：
- 验收日期：
- 收样地点：
- 验收人：
- 对应方向：紫铜手冲/咖啡壶同品
- 固定前提复核：
  - [ ] 仍不是银内壁款
  - [ ] 本次仅为观察样，不代表必然上架
  - [ ] 本次结论不阻塞主线

## A. 基础外观
- [ ] 外观是否与现有图文资料一致
- [ ] 是否存在明显划伤、凹陷、色差、氧化异常、污渍
- [ ] 壶身比例、把手比例、上手视觉是否协调

## B. 内壁与材质状态
- [ ] 内壁状态是否与既有描述一致
- [ ] 未出现“银内壁”误判空间
- [ ] 食品接触部位未见异常脱层、异味、可疑涂层问题

## C. 做工一致性
- [ ] 焊点是否均匀、牢靠、无明显粗糙外溢
- [ ] 壶嘴、把手、壶身连接处是否平顺
- [ ] 壶盖、壶口、底部是否存在明显歪斜或装配问题

## D. 功能与手感
- [ ] 控流是否稳定，是否明显滴漏/断流/偏流
- [ ] 持握是否顺手，满水与空壶状态下手感是否可接受
- [ ] 倾倒动作中是否存在烫手、打滑、重心异常

## E. 尺寸与重量复核
- [ ] 实测尺寸与资料基线一致，允许偏差在可接受范围
- [ ] 实测重量与资料基线一致，允许偏差在可接受范围
- [ ] 上手比例与目标使用场景匹配

## F. 包装与运输抗损
- [ ] 外包装是否完整
- [ ] 缓冲保护是否足够
- [ ] 是否出现运输变形、压伤、磕碰
- [ ] 现有包装方案是否支持后续复用

## G. 结论
- 验收结论：通过 / 有条件通过 / 不通过
- 核心问题：
- 是否影响未来上架判断：是 / 否
- 建议回传对象：运营总监 / 流量 / 技术 / CEO
```

### 3.3 问题分级标准

| 级别 | 判定标准 | 处理动作 | 是否影响主线 |
| --- | --- | --- | --- |
| `P1 致命` | 明显与既定方向冲突；例如内壁状态与“非银内壁款”前提冲突，或存在严重漏水、结构不稳、明显运输毁损 | 24 小时内回传 CEO 与运营总监，支线结论记为“不通过” | 不影响主线继续，但禁止把样品结论用于上线解锁 |
| `P2 重大` | 控流明显不稳、焊接粗糙、运输变形、包装抗损不足、手感明显不达预期 | 24 小时内回传运营总监与供应链协调，标记需复核 | 不影响主线继续 |
| `P3 一般` | 轻微划伤、轻微色差、轻微尺寸偏差、可记录但不改变方向判断的问题 | 记录在验收表内，汇总到周报 | 不影响主线继续 |
| `P4 观察项` | 可接受的手工痕迹、正常铜色差、需持续观察但不构成风险 | 归档留证 | 不影响主线继续 |

### 3.4 回传格式

统一使用以下格式，避免不同角色收到的信息颗粒度不一致：

```md
# 线下送样回传

- 样品：紫铜手冲/咖啡壶同品
- 到样日期：
- 验收结论：通过 / 有条件通过 / 不通过
- 问题级别：P1 / P2 / P3 / P4
- 结论摘要：用 3 句以内说明控流、做工、包装/运输结果
- 是否影响未来上架判断：是 / 否
- 对运营总监：是否调整商品叙事、上线节奏、素材计划
- 对流量：是否继续预热、是否继续用占位素材
- 对技术：是否解锁 `sample_verified`、实拍图替换、商品从 `draft` 转 `active`
- 附件：验收照片、视频、称重/尺量记录、包装照片
```

## 4. 不阻塞主线规则

### 支线未触发时，主线仍可继续的动作

以下动作不依赖样品，可按当前零送样主线继续推进：

- 商品定位与中文/英文卖点整理。
- `product-schema.json` 中除定价外的结构化资料整理。
- Shopify draft 商品创建与变体结构配置。
- 页面模板推送到 draft 环境。
- 运费模板、规格字段、标签与基础信息录入。
- 明确文案中不出现“银内壁”表述。
- 上线前 `阶段一：零送样可立即完成` 的全部工作。

### 只有样品到手后才允许解锁的动作

以下动作继续维持锁定，直到支线完成并回传：

- 实拍主图与细节图替换。
- 实物颜色/光泽与文案的最终核对。
- 实测重量与规格表复核。
- `sample_verified` 从 `false` 改为 `true`。
- 页面“内部预览”警告移除。
- 商品状态从 `draft` 改为 `active`。

### 支线完成后的回传对象与动作

| 回传对象 | 需要收到的结论 | 收到后动作 |
| --- | --- | --- |
| 运营总监 | 验收结论、问题级别、是否影响未来上架判断 | 决定是否调整商品叙事、节奏与是否继续保留候选 |
| 流量 | 是否继续用当前占位素材预热；是否需要等待实拍 | 调整内容排期，不把样品等待当作投放前置 |
| 技术 | 是否允许解锁样品相关 gating 项 | 仅在验收通过或有条件通过时执行样品相关解锁 |
| CEO | 支线是否验证了核心假设；是否值得进入下一步 | 只做决策参考，不回头阻塞主线已在做的工作 |

## 5. 与现有主线文档的关系

- 主线零送样推进规则，继续以 [pre-launch-gating-checklist.md](./pre-launch-gating-checklist.md) 为准。
- 样前资料与规格基线，继续复用现有供应商与 SKU 核验口径。
- 本文档只定义“何时启动线下送样支线、怎么验收、怎么回传、如何不阻塞主线”，不替代主线流程。
