# 店铺装修与页面体验方案

> 对应任务：[HTH-258](/HTH/issues/HTH-258)  
> 当前阶段：preview-only + notify-only  
> 店铺方向：铜器茶具 / 咖具 / 餐具 Shopify 店铺  
> 核心锚点：非银内壁紫铜水瓶手冲咖啡壶  
> 交付对象：CEO、CTO、运营、流量、商品内容团队

## 一页式 CEO 摘要

本阶段店铺不做正式销售站，而是从单商品预览页升级为一个可信的「资料预览型店铺」。页面目标是让访问者清楚理解三件事：这家店关注铜器茶具 / 咖具 / 餐具；当前只公开一款紫铜水瓶手冲咖啡壶的已知资料；现在唯一可行动作是留下到货通知。

装修方向采用「档案馆式铜器工坊」：克制、留白、材质感明确，不做热闹电商促销。主色用深氧化铜棕、米白纸底、浅砂色和少量铜锈绿点缀；排版以大标题、编号卡片、资料标签和对照清单建立可信边界；图像全部标注为资料图、占位图或待实拍，不伪装成已完成商品图。

最小页面结构建议为：首页、PDP、研究 / 内容页、FAQ / 信任页、到货通知入口、页脚 / 政策入口。首页负责建立店铺不是空壳的第一印象，PDP 负责承接单品资料和 notify-only 转化，研究页解释样品复核与资料边界，FAQ / 信任页集中回答为什么不开售、为什么没有价格和检测，页脚承接政策、阶段说明和联系方式。

CTO 侧需要实现的是可配置 section 与 preview 状态控制，不需要立即开发完整购物体验。核心技术要求是 `sample_verified=false` 时全站隐藏价格、库存、购买按钮、变体选择、评论和发货承诺，只显示状态横幅、资料图标识、待复核字段、FAQ 和到货通知表单。

运营侧需要优先提供 Hero 资料图或安全占位图、店铺一句话、阶段说明、FAQ、政策草稿、图像标注和邮件通知文案。CEO / board 需要拍板的事项是：是否开放店铺给外部访问、是否接入 Shopify Admin API / MCP、是否启用密码保护、是否采用 Wokiee 当前主题继续轻装改造。

## 体验原则

1. 先说明阶段，再展示商品。每个入口首屏必须出现「资料预览 / 样品待验收 / 仅开放到货通知」语义，避免用户误以为已经开售。
2. 用资料感补足空店感。通过研究页、透明说明、资料标签、FAQ 和政策入口，让店铺看起来是一个有边界的预览体系，而不是只有一张商品卡的空壳。
3. 所有承诺向后置。价格、库存、交期、检测、控流表现、评论和正式购买入口都等样品验收与商业决策完成后再解锁。
4. 单 SKU 不伪装成多 SKU。当前只围绕紫铜水瓶手冲咖啡壶，不做容量选择、材质选择、系列导购或猜你喜欢。
5. 图像不冒充实拍。资料图、图册裁切、轮廓图和占位图都可以用，但必须显性标注来源状态。

## 视觉与装修方向

主视觉方向：档案馆式铜器工坊。

这个方向把店铺处理成一组「器物资料卡 + 工坊观察记录」，不是促销货架。它适合当前无样品、无价格、无正式库存的阶段，因为可信感来自透明和克制，而不是销售氛围。

### 色彩

| 用途 | 建议色 | 说明 |
| --- | --- | --- |
| 主背景 | `#F5EFE6` 米白纸底 | 让页面像资料册，降低电商促销感 |
| 主文字 | `#2B211A` 深棕黑 | 保持高可读性和沉稳感 |
| 主按钮 / 强调 | `#4D3727` 氧化铜棕 | 与现有技术文档中的铜器调性一致 |
| 卡片背景 | `#FFF9F1` 暖白 | 用于资料卡、FAQ、说明区 |
| 辅助线框 | `#D8C7B5` 浅砂棕 | 表格、卡片边框、模块分隔 |
| 少量点缀 | `#496B5D` 铜锈绿 | 只用于状态标签、已知 / 待复核区分 |
| 警示提示 | `#8A5A2B` 暖铜橙 | 用于样品待验收提示，不用危险红 |

### 排版

标题应有明显的器物资料感。中文标题可使用宋体 / 仿宋方向，如 Noto Serif SC、Source Han Serif SC 或 Shopify 主题可用的 serif 中文字体；正文使用清晰的中文无衬线，如 Noto Sans SC。不要用默认英文字体堆砌，也不要把页面做成科技 SaaS 风。

页面节奏建议为大留白、窄正文、宽资料卡。首页首屏可以大标题 + 小编号标签，下面用 3 到 4 张资料卡建立锚点。规格、FAQ 和透明披露使用表格 / 对照清单，不用营销长段落。

### 图像风格

当前允许三类图像：

| 类型 | 使用位置 | 标注方式 | 禁止处理 |
| --- | --- | --- | --- |
| 图册主图 / 裁切图 | Hero、PDP 媒体区、器型说明 | 角标「资料图」 | 禁止修成实拍质感，禁止去掉资料来源语义 |
| 轮廓图 / 线稿 | 首页器型说明、研究页 | 图注「轮廓示意」 | 禁止暗示实物比例已复核 |
| 材质氛围图 | 背景纹理、模块分隔 | 图注「材质氛围，不代表实物」 | 禁止用作商品实拍 |

### 如何避免「空店 + 单商品壳」

1. 首页不只放商品 Hero，还要放「为什么现在只开放通知」「已知资料 / 待复核」「研究记录」「FAQ 摘要」。
2. 导航不只保留商品链接，还要有「研究记录」「当前阶段说明」「到货通知」。
3. 页脚必须出现政策入口和阶段说明入口，即使政策内容是预览态说明，也比空页脚可信。
4. PDP 下方要有透明披露和 FAQ，不把单品页做成缺价格的残缺购买页。
5. 研究 / 内容页用来解释样品复核流程、资料来源和后续更新节奏，让用户理解店铺正在建设中而不是未完成。

### 状态控制要求

`sample_verified=false` 时：

| 模块 | 处理方式 |
| --- | --- |
| 购买按钮 | 隐藏，不保留 disabled Add to Cart 文案 |
| 价格 | 隐藏真实价格，显示「价格待样品验收后公布」 |
| 库存 | 隐藏库存数字、缺货、售罄、低库存等状态 |
| 交期 | 不显示任何天数或即将发货 |
| 变体选择 | 隐藏，改为单 SKU 锁定说明 |
| 评论 | 隐藏，不显示空评论模块 |
| 推荐商品 | 隐藏，避免多 SKU 误解 |
| 结构化数据 | 不输出价格、库存、offer 可售结构化数据 |
| CTA | 只显示到货通知 |

## CTO 接入清单

P0：

1. 保持 `sample_verified=false` 熔断逻辑，确认 PDP 不输出购买按钮、价格、库存、变体和可售结构化数据。
2. 将首页模块调整为：状态条、Hero、资料卡、器型观察、透明说明、研究记录预告、FAQ、通知 CTA。
3. 增加或复用可配置的资料图角标能力，至少支持「资料图」「占位图」「待实拍」三种。
4. 配置 `/pages/faq` 和 `/pages/research-notes` 的最小页面模板或 Shopify 页面入口。
5. 设置导航：`首页`、`紫铜咖啡壶`、`研究记录`、`到货通知`；页脚：`当前阶段说明`、`FAQ`、`隐私政策`。
6. 到货通知表单优先用 Shopify customer form，tag 使用 `notify-copper-bottle-pour-over-kettle`。

P1：

1. 把 FAQ、透明说明、规格字段尽量做成主题编辑器可配置项，避免运营改 Liquid。
2. 接入 GA4 并将到货通知提交标记为 lead / conversion。
3. 若启用 Klaviyo，保留 Shopify 原生表单 fallback。
4. 为研究页提供通用内容模板：标题、正文、资料卡、CTA。

P2，样品后再做：

1. 实拍图库、细节视频、体验笔记。
2. 价格、库存、发货和正式购买流程。
3. 评论、推荐商品、多 SKU 或集合导购。

## 运营素材清单

P0 必需：

1. 店铺一句话：一句说明本店当前关注铜器茶具 / 咖具 / 餐具，但首期只开放紫铜水瓶手冲咖啡壶资料预览。
2. Hero 资料图：图册主图或安全占位图 1 张，必须允许标注「资料图」。
3. 器型图：图册裁切图、轮廓图或可接受纯文字占位。
4. FAQ 最小 8 条：是否可买、是否银内壁、是否单 SKU、是否有实拍、价格是否确定、何时到货、规格是否确定、邮箱用途。
5. 研究页正文：为什么现在只开放通知、哪些信息待复核、样品后会补什么。
6. 通知表单文案：标题、说明、按钮、提交成功提示。
7. 页脚阶段说明：当前不开售、不预订、不锁库存。
8. 隐私 / 邮箱用途说明：邮箱只用于到货通知和相关更新。

禁止运营提供或上墙：

1. 银内壁、999 银、镀银内壁相关表述。
2. 食品级、食品安全、检测通过、符合标准等结论。
3. 精准控流、稳定萃取、提升风味、专业级、比赛级等性能型卖点。
4. 现货、即发、预售、预约、候补、限量、锁定名额。
5. 多容量、多把手、多 SKU 选择。
6. 未锁定产地、手工锻打等工艺叙事。

## 流量 / 内容投放边界

当前可以引流到首页、研究页或 PDP，但流量文案只能使用「资料预览」「到货通知」「样品验收后更新」。不要把到货通知写成预约、候补、预售或内测名额。

适合的流量钩子：

1. 一款紫铜水瓶手冲咖啡壶的资料预览。
2. 非银内壁版本，样品待验收。
3. 为什么我们先开放到货通知，而不是直接开售。
4. 样品到手后会补齐实拍、规格复核和定价。

不适合的流量钩子：

1. 限时开售、限量抢先看、预售排队。
2. 食品安全、检测结果、控流表现。
3. 专业咖啡师、比赛级、提升风味。
4. 价格优势、现货库存、发货承诺。

## 执行顺序

1. CEO / board 拍板公开范围、API / MCP 接入、主题继续轻改。
2. CTO 固化 preview 状态控制和首页 / PDP / FAQ / 研究页入口。
3. 运营补齐 P0 文案、图像、FAQ、阶段说明和邮箱用途说明。
4. CTO 挂载素材并验证全站无价格、库存、购买、交期和评论模块。
5. 流量团队只用资料预览和到货通知口径做小范围引流。
6. 样品验收后，再由 CEO 决定是否进入正式销售体验设计。
