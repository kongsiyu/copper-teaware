# Capsule Preview 商业闭环与内容结构简报

> 对应任务：[HTH-225](/HTH/issues/HTH-225)
> 当前阶段：Capsule Preview / Pre-launch Foundation
> 结论状态：已完成，综合 [HTH-232](/HTH/issues/HTH-232)、[HTH-233](/HTH/issues/HTH-233)、[HTH-234](/HTH/issues/HTH-234)、[HTH-235](/HTH/issues/HTH-235)

## 当前固定前提

- 当前不是正式销售型店铺上线，而是 `可预览外放 + 收集兴趣信号` 的前置阶段。
- 核心商品锚点仍只保留：`非银内壁紫铜水瓶手冲咖啡壶`。
- 不新增询盘。
- 不新增供应商搜索。
- 不下单。
- 不支付样品费。
- 不扩成多 SKU 正式上架。
- 不把该款写成银内壁款。

## 本单回答的 4 个问题

1. 当前 Capsule Preview 至少需要什么内容结构，才不至于像“空店 + 单商品占位”？
2. 当前市场公开样本是否支持这件商品继续进入预览态推进？
3. 在不下单、不送样的前提下，能否做出可用于 go/no-go 的最小定价、物流与利润判断？
4. 哪些内容可以进入预览态页面，哪些只能内部保留，哪些必须禁止写？

## 1. 结构收口：1 + 2 + 2

### 1.1 收口判断

- 不能只是“一个商品 + 一个空首页”。
- 结构必须同时解决：空店感、为什么值得看、为什么现在不卖、为什么可以留下联系方式。
- 每个锚点都必须映射到实际页面模块，而不是只停留在概念层。

### 1.2 最终结构

本次收口结论：Capsule Preview 不扩成“品牌故事 + 多栏目空架子”，而是压缩为 `1 个核心商品锚点 + 2 个内容锚点 + 2 个信任锚点`。

页面顺序应当是：

`商品识别 -> 内容理解 -> 内容理解 -> 信任确认 -> 信任确认 -> 到货通知`

| 锚点 | 解决什么问题 | 准备放在哪个页面/模块 | 需要什么输入 | 当前能否在预览态外放 |
| --- | --- | --- | --- | --- |
| `1 个核心商品锚点`：非银内壁紫铜水瓶手冲咖啡壶 | 避免页面被理解成空店占位或多 SKU 壳子，让用户一眼记住当前唯一推进对象 | 首页 Hero、PDP 标题区、单 SKU 锁定说明、私域首屏图文 | 产品名、Hero 资料图或占位图、`非银内壁版本` 标签、`单一 SKU` 说明 | 可以，且必须外放 |
| `内容锚点 1`：器型识别与低承诺使用场景 | 回答“为什么值得看”，给页面可转述的记忆点，但不偷渡性能承诺 | 首页器型说明、Image + text 模块、PDP 描述区、社媒首条图文 | 水瓶型轮廓、紫铜壶体视觉特征、手冲咖啡/咖啡角陈列/礼赠候选等低承诺场景文案、图册裁切图 | 可以，但只能写器型、材质、场景，不能写控流、风味、专业级表现 |
| `内容锚点 2`：已知规格与单品事实卡 | 回答“现在到底确认了什么”，避免资料页只有主图和 CTA，没有可核对信息 | 首页已知信息卡、PDP 规格表、FAQ、透明说明区的已知信息列 | 容量 500 cc、口径 5 cm、底径 6.1 cm、高度 15.6 cm、壶盖紫铜、把手/盖钮电木；全部带 `资料显示 / 待样品复核` | 可以，但每一条都必须带待复核标记 |
| `信任锚点 1`：已知资料 / 待样品复核的透明披露 | 回答“为什么现在不能把它当正式商品页”，主动控制预期 | 首页透明说明区、PDP 状态横幅、FAQ、落地页透明披露模块 | `已知资料` / `待复核` 两列清单、`样品待验收` 提示、资料图/占位图角标 | 可以，且必须外放 |
| `信任锚点 2`：notify-only 交易边界 | 回答“用户现在能做什么”，把转化动作收敛成低承诺兴趣收集 | 首页 CTA、PDP 通知表单、页头提示条、表单提交后确认文案 | `到货通知` CTA、`sample_verified=false` 熔断、隐藏价格/库存/加购入口、表单说明文案 | 可以，且必须外放 |

### 1.3 最低成立线

- 如果页面只有一张主图、一个产品名和一个邮箱表单，没有规格事实卡或透明披露，它仍然只是 `单商品占位页`，不算 Capsule Preview。
- 如果页面能讲器型与场景，但没有 notify-only 交易边界，用户会误以为这是一个故意不写价格的销售页，信任仍然不成立。
- 如果页面只有限制条件，没有核心商品锚点和内容锚点，用户无法建立记忆点，也不会留下兴趣信号。

## 2. 最小竞品样本：12 个

### 2.1 样本用途

- 四类样本必须都有，避免只看低价平台导致整体带判断失真。
- 样本的作用是确认“市场带与表达带”，不是证明销量。
- 竞品样本必须能反推三件事：预览态内容口径、价格带天花板、信任表达方式。

详细样本见：`docs/capsule-preview-competitive-benchmarks.md`

### 2.2 市场带结论

- 低价平台主带：`$9-$40`，以不锈钢或铜色外观功能壶为主，不适合作为本 SKU 的核心对标。
- Etsy 手工带：`$60-$130`，能承接“手作 / 铜器视觉 / 复古器型”表达，但经常使用 `tin lined`、free shipping、手工产地等信任要素，本 SKU 不能直接借用。
- 高端手工 / 品牌独立站带：`$112-$150+`，由 Hario、Kalita、KINTO 一类品牌或买手店承接，价格成立依赖更强品牌信任。
- 批发源头带：`$2-$6`，主要是 B2B 供货锚点，只能说明 commodity 成本下沿，不能反推前台售价。

### 2.3 对本 SKU 的含义

- 当前公开样本支持这款 `非银内壁紫铜水瓶手冲咖啡壶` 继续进入 Capsule Preview。
- 但它的可比带更接近 `中档手工 / 设计感铜色咖啡器具`，不是低价平台 commodity，也不是已验证高端手工品牌。
- 内部默认参考带应收在 `$109-$139`，前台页面仍建议不展示价格，只保留 `到货通知 / 样品待验收 / 价格待确认`。

## 3. 最小商业闭环

### 3.1 评审标准

- 三档售价至少区分：保守试探价、主推锚点价、上沿测试价。
- 两条物流路线至少要回答：哪条更适合 Capsule Preview 初期试探，哪条只能留待后置。
- 允许保守占位值，但必须明确哪些数字是内部假设，不能当作对外承诺。

详细测算见：`docs/capsule-preview-fulfillment-model.md`

### 3.2 三档售价假设

| 售价带 | 当前判断 | 原因 |
| --- | --- | --- |
| `$119` | `No-Go` | 两条路线都偏紧，包邮与最低获客试探空间不足 |
| `$149` | `Conditional Go` | 只对 `中国直发美国` 勉强成立，可作为当前最小可行起点 |
| `$179` | `Conditional Go` | 两条路线都开始成立，但需要更强信任表达与更真实履约报价支撑 |

### 3.3 两条物流路线

| 路线 | 当前判断 | 结论 |
| --- | --- | --- |
| 中国直发美国 | 最小可行 | 更适合预览态、小单量、notify-only 验证，不先背 3PL 固定成本 |
| 美国仓 / 3PL | 当前后置 | 只有当真实仓配报价更优、或售价能稳定站到 `$179`、或需求更明确时才应进入下一阶段 |

### 3.4 当前最小判断

- 以当前样品价、1kg 毛重、23cm 级包装的保守占位测算，`$119` 基本不成立。
- `中国直发美国 + $149` 是当前唯一还算成立的最小试探组合。
- 这意味着当前可以继续做 `Capsule Preview`，但不能把它误解成“已经具备正式售卖模型”。

## 4. 页面与对外红线

继续继承并引用：

- `docs/coffee-kettle-zero-sample-cross-review-one-pager.md`
- `docs/coffee-kettle-zero-sample-lock-pack.md`
- `docs/coffee-kettle-zero-sample-ia-content-pack.md`

### 4.1 可进入预览态页面

- 单一核心商品
- 图册规格待复核
- 资料图 / 占位图
- 到货通知
- 研究与透明说明

### 4.2 只允许内部研究

- 样品价
- 包装尺寸
- 冲突重量
- 供应商视频
- 未核验履约口径
- 保守 landed cost 和 break-even CAC 占位值

### 4.3 绝对禁止写

- 银内壁、镀银、999 银
- 食品安全承诺、检测通过
- 精准控流、专业级、比赛级
- 现货、即发、正式价格承诺
- 正式交付承诺
- 多 SKU 可售表述

## 5. 最终 go / no-go 判断

### 5.1 一句话结论

`Conditional Go`

### 5.2 为什么

- 内容结构已经足以支撑预览态外放，不再只是“空店 + 单商品壳”。
- 市场上存在中档手工 / 设计感铜色咖啡器具参考带，但信任表达不能越界到高端品牌或已验证性能。
- 履约框架至少存在 `中国直发美国 + 中档售价假设` 这一条最小可试探路径，但还不足以支持正式销售判断。

### 5.3 如果继续，下一步顺序

1. 完成技术侧 6 项最小配置，把 Wokiee 主题推进到可预览外放。
2. 把 `1+2+2` 内容与信任结构挂进首页与 notify-only PDP。
3. 以低承诺方式外放预览页并收集兴趣信号，不公开承诺价格、库存、交期。
4. 只有当兴趣信号和后续真实报价更明确后，再决定是否进入样品验证或下一阶段履约深化。

### 5.4 如果不继续，阻断点是什么

当前不是立刻 `No-Go`，但后续若出现以下任一情况，应立即停止继续推进：

- 公开市场的价格接受带明显低于 `$149`
- 真实跨境物流与批量成本显著高于当前保守占位
- 页面最终仍然只能做成“空店 + 单商品壳”，无法形成足够信任结构

## 6. CEO 可直接引用的短结论

当前这条 `Capsule Preview` 路线值得继续，但只能按 `Conditional Go` 推进，而不是当成正式开卖项目。原因很明确：内容结构已经能把“这是什么、为什么值得看、为什么现在不能卖、用户现在能做什么”讲完整；市场上也存在中档手工 / 设计感铜器咖啡壶的参考带；但在保守履约测算下，`$119` 不成立，`$149` 只对 `中国直发美国` 勉强成立，`美国仓 / 3PL` 还不该现在进入。所以最合理的下一步不是上架卖货，而是先把预览态页面做完整，用 `资料预览 + 到货通知` 收集兴趣信号，再决定是否进入下一阶段。
